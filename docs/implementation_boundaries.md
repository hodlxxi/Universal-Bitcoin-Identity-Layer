# Implementation Boundaries

## Purpose
Set explicit limits so HODLXXI documentation remains accurate, conservative, and operationally honest.

## Boundary statements
- HODLXXI does not create universal trust truth.
- HODLXXI does not guarantee trust portability across all domains.
- Key ownership proofs do not replace social verification.
- Descriptor Pruning remains local governance behavior.
- Symbolic time-lock commitments are not sufficient standalone security.

## Operational limits
- Weak Sponsor discipline weakens Trust Tree quality.
- Infrequent interaction reduces observability quality.
- Poorly documented governance increases coordination ambiguity.
- Different domains can reach incompatible but valid local decisions.

## Non-canonical implementations
Implementations may differ in:
- UI/UX,
- data schemas,
- storage backends,
- policy workflows,
- onboarding controls.

Variation is acceptable when the core principles and local-accountability model are preserved.
