# Governance

## Purpose
Define governance expectations for HODLXXI as local, accountable coordination rather than centralized control.

## Governance unit
The Trust Domain is the primary governance unit. Policies can vary between domains while preserving shared principles.

## Governance baseline
- preserve Right to Exit
- keep Sponsor/Edge accountability visible
- document local norms clearly
- prefer branch separation over coercive closure when conflicts are deep

## Forkability over voting
Majority voting can be useful for local operations, but foundational disagreement should resolve through forkability and voluntary re-alignment.

## Practical governance artifacts
A domain may maintain:
- Sponsor expectations
- Branch review routines
- interaction and reciprocity norms
- Descriptor Pruning criteria
- onboarding/offboarding procedures

These artifacts are local and non-canonical.
