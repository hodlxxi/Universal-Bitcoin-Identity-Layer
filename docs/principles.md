# Principles

## Purpose
State the stable principles that define HODLXXI regardless of implementation details.

## 1) Right to Exit
Participants and subgroups can leave, split, or fork without requiring universal approval.

## 2) No Canonical Implementation
HODLXXI is a framework. Different stacks, UX layers, and storage patterns can be valid if they preserve the core model.

## 3) Symmetry of Observability
Trust claims should be anchored in interactions that relevant counterparties can observe over time.

## 4) Forkability Over Voting
Irreconcilable disagreements are better handled by branch separation than forced unanimity.

## 5) Metric Non-Reduction
Reputation is **observable interaction history**. It is not reducible to a single trust scalar.

## 6) Sponsorship Accountability
A Sponsor is reputationally accountable for introduced Edges and, recursively, for Branch quality signals.

## 7) Time as Commitment
Long-horizon commitment structures (including optional symbolic lock patterns) emphasize continuity over short-term extraction.

## Working interpretation
These principles are intended to:
- keep governance local,
- keep accountability legible,
- preserve context,
- and maintain voluntary coordination.
