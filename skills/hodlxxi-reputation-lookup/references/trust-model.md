# Reputation trust framing

Use this framing exactly when summarizing the reputation surface:

`public_key + operator_binding + optional_time_locked_capital + signed_behavior + observable_service_history`

## Boundaries

- `public_key` identifies the runtime's cryptographic anchor.
- `operator_binding` is declared metadata, not independent proof of organizational control.
- `optional_time_locked_capital` is optional and must not be treated as proven unless evidence exists.
- `signed_behavior` comes from signed receipts and similar runtime artifacts.
- `observable_service_history` comes from public history the runtime exposes.
